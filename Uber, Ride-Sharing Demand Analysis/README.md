# Uber Ride-Sharing Demand Analysis

## Project Overview
This project analyzes Uber ride-sharing data to understand ride demand patterns,
peak usage hours, fare distribution, and revenue trends.

The goal of the analysis is to identify usage patterns that can help
transportation companies optimize operations and improve service availability.

## Dataset
Dataset used: Uber Trips Dataset (50K rides)

Columns include:
- trip_id
- city
- distance_km
- fare_amount
- pickup_time
- drop_time

## Tools Used
- Python
- Pandas
- NumPy
- Jupyter Notebook

## Analysis Performed
The following analysis was conducted:

- Daily ride distribution
- Monthly ride demand trends
- Peak ride hours analysis
- Busiest cities analysis
- Average fare calculation
- Revenue analysis per city
- Outlier detection in fares

## Key Insights
- Peak ride demand occurs during evening hours (around 6 PM).
- Some cities generate significantly higher ride volume.
- Average fare is around $16 per ride.
- Ride demand patterns change across different times of the day.
